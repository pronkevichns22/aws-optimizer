   ```

   ```

## ❓ Часто задаваемые вопросы

| Вопрос | Ответ |
|--------|-------|
| **Могу ли я продолжать работу после сдачи?** | ✅ Да! Ваша копия остаётся у вас. Преподаватель создаёт отдельную копию для проверки. |
| **Нужно ли делать Fork или Transfer?** | ❌ Нет. Просто добавьте преподавателя в Collaborators. |
| **Что если я забыл закоммитить файл?** | Используйте `git add .` и `git commit -m "..."`, затем `git push origin main` |
| **Увидит ли преподаватель мои будущие изменения?** | ❌ Нет. Он копирует репозиторий на момент сдачи. Дальнейшие коммиты его не затрагивают. |
| **Можно ли сдать из приватного репо?** | ✅ Да, если добавить преподавателя в Collaborators. |
| **Что если в .env остались секреты?** | ❌ Это критическая ошибка! .env должен быть в .gitignore. Используйте `.env.example`. |
| **Нужно ли удалять node_modules перед сдачей?** | ✅ Да! Они в .gitignore и восстановятся при `npm install`. Это сокращает размер репо. |

## 🔍 Финальная проверка (для вас)

Перед сдачей запустите эту команду:

```bash
# Проверка git
git status                                    # Не должно быть changes
git log --oneline | head -5                  # История коммитов

# Проверка структуры
ls -la                                       # Видны: README.md, .gitignore, docker-compose.yml
ls -la server/                                # Видны: package.json, src/, .env.example
ls -la client/                                # Видны: package.json, src/, .env.example
ls -la docs/                                  # Видны: ARCHITECTURE.md, API.md, SETUP.md и т.д.

# Проверка .gitignore
cat .gitignore | grep -E "^node_modules|^\.env|^\.idea"

# Проверка .env (не должно быть в git!)
git ls-files | grep "\.env$"                 # Не должно быть вывода
```
