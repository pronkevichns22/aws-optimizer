```markdown
### 2.1.1 Почему именно 3-слойная архитектура?

Альтернативы, которые мы отклонили:

1. **Монолитная архитектура в одном файле**
   - Минус: 10,000+ строк в одном файле → невозможно поддерживать
   - Минус: Изменение одного компонента требует перебилдинга всего

2. **Микросервисная архитектура (10+ сервисов)**
   - Плюс: Максимальная масштабируемость
   - Минус: Сложная оркестрация, сложная отладка, увеличенная латентность из-за сетевых вызовов
   - Минус: Для учебного проекта избыточна

3. **Наш выбор: Модульная монолитная архитектура**
   - Это компромисс между простотой разработки и возможностью эволюции
   - Внутри монолита ясно разделены: Auth Module, Rules Module, AI Module
   - Каждый модуль можно позже выделить в микросервис без переписывания кода

Это важно, потому что при масштабировании можно пойти по пути:
Монолит → Модульный монолит → Микросервисы

Без правильной архитектуры на этапе монолита, переход в микросервисы требует полной переписки.
```
